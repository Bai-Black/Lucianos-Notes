# 前言

## 什么是消息中心

“**C++消息中心（Message Center / Message Bus / Event Center）**”并不是 C++ 语言自带的一个标准组件，而是一种**常见的软件架构设计模式**。在嵌入式、游戏引擎、GUI框架、服务器程序中都非常常见。

简单理解：

> **消息中心 = 一个负责“模块之间通信”的中间枢纽**

它让不同模块**不需要直接调用彼此函数**，而是通过发送和接收消息进行通信，从而实现**解耦**。

## 为什么需要消息中心

大型项目中，数据的传递极为复杂，如果不使用消息中心，而是直接在模块间相互调用，可能导致：

- 模块**耦合严重**， 修改一个模块可能影响多个模块
- 不方便扩展
- 代码臃肿

使用消息中心，模块只需要：

- **发送消息**
- **订阅消息**

大大简化了数据的传输。

# 典型示例

> 这套代码由 ChatGPT 生成，较为简单基础
>
> **很多实现方法都并非最优选项！**仅供前期学习

``` c++
#include <map>
#include <vector>
#include <functional>
#include <iostream>

enum class MessageType
{
    SENSOR_UPDATE,
    MOTOR_START,
    MOTOR_STOP
};

struct Message
{
    MessageType type;
    void* data;
};


class MessageCenter
{
public:
    using Callback = std::function<void(const Message&)>;

    void subscribe(MessageType type, Callback cb)
    {
        subscribers[type].push_back(cb);
    }

    void publish(const Message& msg)
    {
        for (Callback& cb : subscribers[msg.type])
        {
            cb(msg);
        }
    }

private:
    std::map<MessageType, std::vector<Callback>> subscribers;
};


MessageCenter center;	//创建消息中心

void MotorHandler(const Message& msg)	//待回调的需要数据的函数
{
    //这里处理msg
    std::cout << "Motor Start!\n";
}

int main()
{
    //让MotorHandler函数订阅MOTOR_START消息
    center.subscribe(MessageType::MOTOR_START, MotorHandler);

    Message msg;
    msg.type = MessageType::MOTOR_START;

    center.publish(msg);	//发布MOTOR_START消息
}
```

输出：

```
Motor Start!
```

## `void*`

``` c++
struct Message
{
    MessageType type;
    void* data;
};
```


`void*` 是 C/C++ 中的一种 **指针类型**，叫做：

> **无类型指针（通用指针 / Generic Pointer）**

意思是：

**它可以指向“任何类型的数据”，但自身不知道数据的具体类型。**

特点：

- 可以存放 **任何类型的地址**
- 但 **不能直接解引用**

### 基本用法

```
int a = 10;

void* p = &a;

printf("%d", *(int*)p);
```

步骤：

```
void* p
   ↓
(int*)p
   ↓
*(int*)p
```

意思是：

> “把 p 当成 int 指针再使用”

### 为什么需要 `void*`

用于**通用接口设计**

使得可以发送与接受任何类型的数据、数组

### 缺点

- 不安全，因为编译器**无法检查类型**

现代C++ 中，通常用`template`代替。

## `std::function`

``` c++
	using Callback = std::function<void(const Message&)>;
```

`std::function` 是 C++ 标准库里的一个 **函数包装器**。

它可以存储：

- 普通函数
- lambda
- 成员函数
- 仿函数

统一当作一个 **“可调用对象”** 来使用。

### 基本格式

```
std::function<返回值(参数列表)>
```

也就是说，示例代码中，`Callback` 被定义为了一个接收 `const Message&` ，返回`void`的函数对象类型。

### 使用示例

``` c++

void handleMessage(const Message& msg)
{
    std::cout << msg.type << std::endl;
}
Message msg{1};

//注册回调
Callback cb = handleMessage;

//调用回调
cb(msg);
```

### 为什么要用`std::function`

如果不用它，你可能要写**函数指针**：

``` c++
void (*callback)(const Message&);
```

问题：

- 不能存 lambda（带捕获）
- 不能存成员函数
- 灵活性差

## `Vector`库

``` c++

```

``` c++
        subscribers[type].push_back(cb);
```

**向量（Vector）**是 C++ 标准库 **STL（Standard Template Library）** 中一个封装了动态大小数组的**顺序容器（Sequence Container）**。跟任意其它类型容器一样，它能够存放各种类型的对象。可以简单的认为，向量是一个能够存放任意类型的**动态数组**。

### 特性

- 顺序序列：顺序容器中的元素按照严格的线性顺序排序。可以通过元素在序列中的位置访问对应的元素。
- 动态数组：支持对序列中的任意元素进行快速直接访问，甚至可以通过指针算述进行该操作。提供了在序列末尾相对快速地添加/删除元素的操作。

### 基本使用

- 定义

``` c++
std::vector<Type> vectorName;
```

- 添加元素x

``` c++
vectorName.push_back(x);
```



示例中：

即创建了一个类内私有的，名为 `subscribers` 的，`key`数据类型为`MessageType的，`value`数据类型为**函数对象的`vector`数组**的`map`对象。

# 队内代码

> message_center.hpp

``` c++
#pragma once
#include <iostream>
#include <unordered_map>
#include <string>
#include <typeindex>
#include <utility>
#include "log.h"
extern "C"
{
#include "FreeRTOS.h"
#include "task.h"
}

// TODO  线程安全(虽然好像没什么问题)

#define pubMessage(topic, data) MessageCenter::pub(topic, data)
#define subMessage(topic, data) MessageCenter::sub(topic, data)

// 自定义哈希函数
namespace std
{
    template <>
    struct hash<std::pair<std::string, std::type_index>>
    {
        size_t operator()(const std::pair<std::string, std::type_index> &key) const
        {
            size_t h1 = hash<std::string>()(key.first);
            size_t h2 = hash<std::type_index>()(key.second);
            return h1 ^ (h2 << 1);
        }
    };
}

class MessageCenter
{
public:
    // 发布消息，使用复合键建图
    template <typename T>
    static void pub(const std::string &topic, const T &data)
    {
        //taskENTER_CRITICAL();
        std::pair<std::string, std::type_index> key(topic, std::type_index(typeid(T)));
        getInstance().messageMap<T>()[key] = data;
        //taskEXIT_CRITICAL();
    }

    // 订阅消息
    template <typename T>
    static void sub(const std::string &topic, T &data)
    {
        //taskENTER_CRITICAL();
        std::pair<std::string, std::type_index> key(topic, std::type_index(typeid(T)));
        auto iter = getInstance().messageMap<T>().find(key);
        if (iter != getInstance().messageMap<T>().end())
        {
            data = iter->second;
        }
        else
        {
            LOGWARNING("MessageCenter: No message found for topic: %s", topic.c_str());
        }
        //taskEXIT_CRITICAL();
    }

private:
    MessageCenter() = default;
    MessageCenter(const MessageCenter &) = delete;
    MessageCenter &operator=(const MessageCenter &) = delete;

    static MessageCenter &getInstance()
    {
        static MessageCenter instance;
        return instance;
    }

    template <typename T>
    using MessageMap = std::unordered_map<std::pair<std::string, std::type_index>, T>;

    template <typename T>
    MessageMap<T>& messageMap()
    {
        static MessageMap<T> map;
        return map;
    }
};
```

## `unordered_map`

`std::unordered_map` 是 C++ 标准库中的一种 **哈希表容器**，定义在头文件`<unordered_map>`

它用于 **通过 key 快速查找 value**，和 `map` 非常类似，但内部实现完全不同。

一句话理解：

> **unordered_map = 基于哈希表的键值对容器，查找速度平均 O(1)**

使用方法与`map`基本相同。

| 特性       | map      | unordered_map |
| ---------- | -------- | ------------- |
| 底层       | 红黑树   | 哈希表        |
| 是否有序   | 有序     | 无序          |
| 查找复杂度 | O(log n) | O(1)          |
| 插入复杂度 | O(log n) | O(1)          |
| 遍历顺序   | 排序顺序 | 随机          |
| 内存占用   | 较小     | 较大          |

## `operator`

`operator` 是一个让你能够为自己的类定义（或重载）运算符的行为的关键字，比如 `+`、`-`、`<<`、`[]`、`()` 等。

使你的 **类对象之间也可以像内置类型一样使用运算符**。

> **C++ 里几乎所有运算符本质都是函数调用。**

### 语法

```c++
返回类型 operator运算符(参数)
```

### 用途

在 C++ 中，很多看起来像 **符号运算** 的东西，本质其实是 **函数调用**。

例如：

```
a + b
```

实际上编译器会理解为：

```
a.operator+(b)
```

或者：

```
operator+(a,b)
```

所以：

| 写法    | 实际含义          |
| ------- | ----------------- |
| `a + b` | `a.operator+(b)`  |
| `a[b]`  | `a.operator[](b)` |
| `a()`   | `a.operator()()`  |

### `operator()`

含义是：

> **重载函数调用运算符 `()`**

#### 语法

```
返回类型 operator()(参数列表) [const]
{
    函数体
}
```

- `const` ：对象是否可被修改

#### 示例

```
#include <iostream>

class Test
{
public:
    int operator()(int x)
    {
        return x * 2;
    }
};

int main()
{
    Test t;

    std::cout << t(5) << std::endl;
}
```

程序执行：

```
t(5)
```

编译器会转换为：

```
t.operator()(5)
```

输出：

```
10
```

#### 用途

在 C++ 中，这种对象叫：

**函数对象（Function Object / Functor）**

优点：

- 可以像函数一样调用

- 可以保存状态

- 可以作为 **模板参数**

## 哈希函数

在 **C++ / 计算机中**，**Hash（哈希）**可以简单理解为：

> **把任意数据，通过一个函数，转换成一个固定大小的数字。**

这个过程使用的函数叫 **哈希函数（Hash Function）**。

### 直观理解

假设有一个函数：

```
hash("apple")  ->  1837291
hash("banana") ->  9283712
hash("cat")    ->  6628193
```

字符串长度不同，但 **hash结果都是固定类型（通常是整数）**。

所以：

**Hash = 数据 → 一个唯一编号（尽量唯一）**

### 主要用途

Hash最重要的用途是 **快速查找**。

例如普通查找：

```
apple
banana
cat
dog
```

找 `"cat"` 需要一个一个比对，时间复杂度为`O(n)`

但如果用 **Hash表（Hash Table）**：

```
hash("cat") → 6628193 → 直接找到
```

时间复杂度为`O(1)`

### 内部实现

`hash`是`C++`通过模板定义的一个类。

### 代码解析

``` c++
// 自定义哈希函数
namespace std
{
    template <>
    struct hash<std::pair<std::string, std::type_index>>
    {
        size_t operator()(const std::pair<std::string, std::type_index> &key) const
        {
            size_t h1 = hash<std::string>()(key.first);
            size_t h2 = hash<std::type_index>()(key.second);
            return h1 ^ (h2 << 1);
        }
    };
}
```

与上文示例使用枚举类型不同，队内代码中将`key`类型设为`std::pair<std::string, std::type_index`，即**消息名称**与**消息数据类型**的数据对。且使用`unordered_map`（性能佳于`map`）。问题在于，`unordered_map`底层依赖**哈希表**，而`std::pair<std::string, std::type_index`作为自定义的数据类型，`C++`内部并没有给它提供**哈希函数**。

**这整段代码，正是通过 模板特化 ，为`std`命名空间里的`hash`类，新增`hash<std::pair<std::string, std::type_index>>`的实现。**

``` c++
size_t h1 = hash<std::string>()(key.first);
size_t h2 = hash<std::type_index>()(key.second);
```

通过新建临时`hash`对象的方式取得默认哈希函数对数据对内两个元素的计算结果

``` c++
return h1 ^ (h2 << 1);
```

通过异或运算取得新的哈希值并返回
